# 接口设计

## 1. 项目定位

本项目用于实现操作系统实验五：简单文件系统。

目标是在内存中模拟一块虚拟磁盘，并在其上实现一个单用户文件系统，支持以下基本功能：

- 文件系统初始化与格式化
- FAT 表管理磁盘空间
- 多级目录管理
- 文件创建与删除
- 文件打开与关闭
- 文件读写
- 文件系统退出与保存

实验要求中明确给出了这些核心功能和函数，并要求小组协作完成，同时每位成员需要能够解释自己负责的函数和整体实现思路。

4. 磁盘布局约定
当前文件系统采用如下磁盘布局：
0 号块：引导块 BLOCK0
1~2 号块：FAT1
3~4 号块：FAT2
5 号块：根目录
6 号块开始：数据区
说明：
根目录固定使用 5 号块
initblock.root = 5
数据区从 6 号块开始
0~5 号块均视为系统保留块，不参与普通文件数据分配
该布局与实验书和往届实现思路一致，便于目录、文件、读写、删除模块统一依赖。
5. 关键常量定义
定义在 include/fs.h 中：
SIZE：虚拟磁盘总大小
END：FAT 链结束标志
FREE：空闲块标志
ROOTBLOCKNUM：根目录初始占用块数
MAXOPENFILE：最多同时打开文件数
DIRLEN：路径最大长度
当前语义约定
END = 65535
FREE = 0
说明：
FAT 中某块值为 FREE 表示该块空闲
FAT 中某块值为 END 表示该块为文件或目录链的最后一块
6. 核心数据结构说明
6.1 fcb
文件控制块，表示一个文件或目录的元信息。
字段说明：
filename[8]：文件名
exname[3]：扩展名
attribute：
0 表示目录
1 表示普通文件
time：创建时间
date：创建日期
first：起始盘块号
length：文件长度
free：
0 表示有效项
1 表示空项或已删除项
6.2 fat
文件分配表项。
字段说明：
id：下一块盘块号
若为 END，表示当前块是链尾
若为 FREE，表示该块空闲
6.3 useropen
用户打开文件表项，用于记录当前已打开文件的运行时状态。
字段说明：
filename / exname：当前打开文件名
attribute：文件属性
time / date：文件时间信息
first：起始块号
length：文件长度
dir[DIRLEN]：当前文件完整路径
count：读写指针
fcbstate：
1 表示内存中的 FCB 已修改，需要回写
0 表示未修改
topenfile：
1 表示该打开表项已占用
0 表示空闲
dirno：父目录所在盘块号
diroff：当前文件在父目录中的偏移
open_fcb：当前文件对应的 FCB 副本
6.4 block0
引导块，保存文件系统元信息。
字段说明：
information[200]：描述信息
root：根目录起始盘块号
startblock：数据区起始地址
blocksize：块大小
blocknum：块数量
7. 路径与目录约定
当前路径格式
当前目录路径统一使用：
根目录：~/
子目录路径示例：~/dir1/
更深层路径示例：~/dir1/dir2/
目录内容约定
目录本质上也是文件，但其内容不是普通文本，而是一组目录项（FCB 列表）。
目录创建时，默认应包含两个特殊目录项：
.
..
因此对于目录：
初始长度通常为 2 * sizeof(fcb)
删除目录时，判断是否为空目录时，必须忽略 . 和 .. 这两个特殊项。
这是实验中非常关键的逻辑点。
