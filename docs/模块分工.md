# 模块分工

## 1. 分工原则

本项目采用“先统一公共框架，再分模块并行开发”的方式协作完成。

目前组长已完成以下基础工作：

- 仓库结构搭建
- Makefile 编译配置
- `.gitignore` 配置
- `README.md` 编写
- 公共头文件与核心结构体定义
- `utils.c` 公共辅助函数初版
- `fs.c` 中 `startsys()`、`my_format()`、`my_exitsys()` 基础版
- `main.c` 命令循环骨架
- 磁盘布局、路径格式、字段语义统一约定

因此当前项目已经具备并行开发条件，各成员在统一接口基础上完成各自模块。

---

## 2. 当前统一约定

### 2.1 磁盘布局

- 0 号块：引导块
- 1~2 号块：FAT1
- 3~4 号块：FAT2
- 5 号块：根目录
- 6 号块开始：数据区

### 2.2 关键字段语义

- `attribute = 0`：目录
- `attribute = 1`：普通文件
- `free = 0`：有效项
- `free = 1`：空项或已删除项
- `count`：读写指针
- `fcbstate = 1`：FCB 已修改，需要回写
- `topenfile = 1`：打开文件表项已占用

### 2.3 路径格式

- 根目录：`~/`
- 子目录示例：`~/dir1/`

## 3. 小组成员分工

| 成员 | 主要负责模块 | 主要函数/内容 | 主要任务说明 |
|---|---|---|---|
| 组长 | 公共框架 + 系统初始化退出 + 总体联调 | `startsys()`、`my_format()`、`my_exitsys()`、公共头文件、公共辅助函数、命令循环 | 负责整个项目的公共底层与接口统一，维护可运行骨架，负责最终联调与整体整合 |
| 组员1 | FAT 与磁盘空间管理 | `getFreeFatid()`、`fatFree()`、`getNextFat()` 等 | 负责 FAT 表相关逻辑，包括空闲块查找、块链分配、块链释放和下一块查找，为创建、删除、读写提供底层支持 |
| 组员2 | 目录浏览与切换 | `my_ls()`、`my_cd()` | 负责目录内容读取、显示与当前目录切换，正确维护路径与当前目录状态 |
| 组员3 | 目录创建与删除 | `my_mkdir()`、`my_rmdir()` | 负责子目录创建和删除，正确处理 `.`、`..` 目录项以及空目录判断 |
| 组员4 | 文件创建、删除、打开、关闭 | `my_create()`、`my_rm()`、`my_open()`、`my_close()` | 负责文件控制流程，包括创建文件、删除文件、打开文件、关闭文件，以及目录项与打开文件表的维护 |
| 组员5 | 文件读写 + 测试整理 | `my_write()`、`do_write()`、`my_read()`、`do_read()` | 负责普通文件的读写逻辑，包括追加写、覆盖写、截断写、跨块读写，并协助整理联调测试记录 |

## 4. 各成员需要重点掌握的内容

### 组长
需要重点掌握：
- 虚拟磁盘整体结构
- 文件系统初始化流程
- FAT、根目录、数据区的布局
- 为什么退出时需要保存整个虚拟磁盘
- 各模块如何衔接联调

### 组员1
需要重点掌握：
- FAT 表的作用
- `FREE` 与 `END` 的含义
- 文件块链如何建立和释放
- 删除文件或目录时如何回收整条块链

### 组员2
需要重点掌握：
- 目录为什么本质上也是文件
- `my_ls()` 如何读取目录项
- `my_cd()` 如何切换当前目录
- 路径如何维护

### 组员3
需要重点掌握：
- 创建目录时为什么要写入 `.` 和 `..`
- 删除目录前为什么必须判空
- 如何在父目录中写入新目录项

### 组员4
需要重点掌握：
- 文件创建时如何检查重名
- 如何分配首块并写入目录项
- 打开文件为什么需要打开文件表
- 关闭文件时为什么需要回写 FCB

### 组员5
需要重点掌握：
- 读写指针如何维护
- `my_write()` 与 `do_write()` 的区别
- `my_read()` 与 `do_read()` 的区别
- 追加写、覆盖写、截断写的区别
- 跨块读写如何处理

---

## 5. 每位成员的开发要求

1. 在统一接口和结构体基础上完成自己模块，不得随意修改公共结构体和函数原型  
2. 若确实需要修改公共接口，必须先与组长沟通，再统一调整  
3. 每位成员除了完成自己模块代码外，还需补充自己模块的测试用例  
4. 每位成员应能说明自己模块与其他模块的依赖关系  
5. 每位成员在验收前应能独立讲清自己负责函数的实现思路和关键代码

---

## 6. 模块开发建议顺序

### 第一阶段：组长先完成公共底层
- 公共结构体
- 公共接口
- `utils.c`
- `fs.c`
- `main.c`

### 第二阶段：并行开发
- 组员1：FAT 模块
- 组员2：目录浏览与切换
- 组员3：目录创建与删除
- 组员4：文件创建、删除、打开、关闭
- 组员5：文件读写与测试整理

### 第三阶段：统一联调
建议联调顺序：

1. `my_format()`
2. `my_ls()`
3. `my_mkdir()` / `my_cd()`
4. `my_create()` / `my_open()`
5. `my_write()` / `my_read()`
6. `my_close()`
7. `my_rm()` / `my_rmdir()`
8. `my_exitsys()`

## 7. 当前结论

当前项目基础框架已完成，已经具备并行开发条件。  
后续每位成员按照本分工文档认领模块，在统一接口与磁盘布局约定下完成自己的部分，并补充测试内容。

最终验收时，需要说明分工情况，并保证每位成员都能够解释自己模块的核心函数与整体实现思路。
